## Fixes Applied (Template-Level)

The following issues have been fixed in this branch:

### 1. Broken skip-to-content link (8 pages)

**Error:** `G1,G123,G124.NoSuchID` — skip nav link pointed to `#content` but the target element has `id="main-content"`.

**File:** `view/layout/layout.phtml`
**Fix:** Changed `href="#content"` to `href="#main-content"`.

### 2. Empty menu-backer link (8 pages)

**Error:** `A.NoContent` — the mobile menu close button `<a id="menu-backer">` had no accessible text content. The `$this->translate()` calls were missing `echo`, so nothing was rendered.

**File:** `view/common/menu-drawer.phtml`
**Fix:** Added `echo` to translate calls, added `aria-label` attribute, added screen-reader-text span for accessible content.

### 3. Empty `lang=""` attribute on metadata values (55 errors across 4 pages)

**Error:** `WCAG2AA.Principle3.Guideline3_1.3_1_2.H58.1.Lang` — `<dd>` elements rendered with `lang=""` when no language was set for a metadata value, which is an invalid language tag.

**File:** `view/common/resource-values.phtml`
**Fix:** Only render the `lang` attribute when `$valueLang` is non-empty.

### 4. Image-only links missing alt text (120 errors across 4 pages)

**Error:** `H30.2` — thumbnail links using `linkRaw()` wrapped images with no alt text, providing no accessible name for the link.

**Files:**
- `view/omeka/site/item/browse.phtml`
- `view/omeka/site/item-set/browse.phtml`
- `view/common/block-layout/browse-preview.phtml`
- `view/common/block-layout/item-showcase.phtml`

**Fix:** Added `aria-label` with the resource's display title to each `linkRaw()` call.

## Requires Designer Input: Color Contrast

The following color contrast failures need review from the designer. All fail the WCAG 2.1 AA minimum contrast ratio of **4.5:1** for normal text.

### Golden navigation dropdown links (79 errors, all pages)

**Error:** `G18.Fail` — white text (`#FFFFFF`) on gold background (`#EDAA16`).

**Location:** `nav.golden-navigation` dropdown menu links
**SCSS file:** `asset/sass/components/navigation/_navigation.scss` (line 316)
**Current:** `color: $color__white` (`#FFFFFF`) on `background: $color__gold` (`#EDAA16`)
**Contrast ratio:** ~1.94:1 (fails 4.5:1)

**Options for the designer:**
- Darken the text to `#333333` or similar dark color on the gold background (ratio ~7.3:1)
- Darken the gold background to at least `#8B6600` if white text is preferred (ratio ~4.6:1)
- Use a different background/text color combination that achieves 4.5:1

### Mega menu links (secondary-light on purple)

**Location:** `.main-navigation .navigation > li > ul a`
**SCSS file:** `asset/sass/components/navigation/_navigation.scss` (line 209)
**Current:** `color: $color__secondary-light` (`#eef7fa`) on `background: rgba($color__purple, 0.96)` (`#612A60`)
**Note:** This may actually pass (light text on dark background), but pa11y flags it due to the `rgba` transparency preventing automated calculation. Recommend manual verification.

### Pagination text

**Location:** `.pagination .row-count`, `.pagination .pager label`, `.pagination .page-count`
**SCSS file:** `asset/sass/components/pagination/_resources-pagination.scss` (lines 33, 41)
**Current:** `color: $color__secondary-text` (`$color__gray` = `#8C8279`) on `background: $color__off-white` (`#F6F4F3`)
**Contrast ratio:** ~2.7:1 (fails 4.5:1)

**Options:**
- Darken the text to `#5C5249` or darker (ratio ~4.6:1)
- Use `$color__gray-33` (`#555`) which would pass (ratio ~6.4:1)

### "Random items" heading text and "See More" button

**Location:** `#randomItems h4` and `.see-more-btn` (rendered via page content/blocks, likely controlled in Omeka admin or page block CSS)
**Note:** These appear to be styled via page content blocks rather than theme templates. The contrast issue is likely inherited from a shared style. Check if these elements use `$color__secondary-text` or a similar light gray.

## Not Fixable in Theme (Omeka Core / Module Issues)

### Iframe missing title (2 errors, item detail pages)

**Error:** `H64.1` — OHMS embed iframes lack `title` attributes.
**Source:** The `<iframe>` is generated by the OhmsEmbed module, not the theme. This should be reported as a bug to the OhmsEmbed module maintainer.

## Warnings (Advisory, Not Failures)

618 warnings were flagged. Most are expected and do not require action:

- **G18.Alpha / G18.Abs** — Transparency and absolute positioning prevent automated contrast checking. Manual spot-checks recommended.
- **H67.2** — Decorative images correctly marked with `alt=""`.
- **H48** — Suggests wrapping link groups in `<ul>`. Already done where appropriate.
- **G141** — Heading hierarchy gaps in CMS content. Low priority for legacy content.

## Limitations

- Automated testing catches approximately 30-40% of WCAG issues. Manual testing with keyboard navigation and screen readers is still recommended.
- Only 8 pages were tested (Omeka S does not generate a sitemap by default). Additional page types (e.g., media show pages, search results) should be tested separately.
- Content-level issues (broken links, empty headings in CMS content) are not addressed by template fixes.
